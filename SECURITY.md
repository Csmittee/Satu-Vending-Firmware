# SATU — Security Architecture
<!-- Read this before touching auth, ownership, payment, or factory reset code -->
<!-- Last updated: 2026-05-31 -->

## Four Security Layers (never mix them)

| Layer | What it protects | Who controls it | Where stored |
|-------|-----------------|-----------------|--------------|
| L1 — Machine physical | Service mode entry | Machine owner | NVS `svc_pin` |
| L2 — Dashboard web | Temple owner portal | Machine owner | D1 `users.password_hash` (PBKDF2) |
| L3 — Device API | Machine ↔ backend calls | We control | NVS `dev_secret` + D1 `devices.device_secret` |
| L4 — Admin API | Cloudflare admin access | We control only | Cloudflare secret `ADMIN_SECRET` |

Owner buying a machine gets L1 + L2. Never L3 or L4. Never tell the owner what device_secret is.

## Machine Identity Chain

```
Factory flash
  → MAC address (ESP32 hardware, immutable)
  → 6-digit setup code (random, generated by backend, stored in D1 setup_codes table)
  → printed on sticker inside lid + box QR

First power-on
  → machine calls POST /v1/machine/hello with MAC
  → backend creates device record, returns status=pending + setup_code
  → machine displays setup code screen

Owner claim flow
  → owner registers at dashboard.satu-th.com (email or LINE)
  → enters 6-digit code → POST /v1/machine/claim
  → backend: verifies code, binds owner_id to device_id, sets status=active
  → machine polls /hello every 30s → gets status=active → shows idle grid

Ownership is permanent until owner releases OR admin nukes.
```

---

## Ownership Rules (AirTag model)

- One machine can only belong to one owner at a time
- New 6-digit code generated after any ownership transfer or factory reset
- Old 6-digit code is invalidated immediately on claim
- Unique constraint on `setup_codes.assigned_mac` — no two machines share a code
- If MAC collision detected on /hello: return existing device_id, do NOT create duplicate

## Suspension vs Nuke vs Factory Reset

| Action | Who initiates | What happens | Machine behaviour |
|--------|--------------|-------------|------------------|
| Suspend | Admin (us) | `devices.status = suspended` | /order returns 403, shows "Service unavailable" |
| Enable | Admin (us) | `devices.status = active` | Normal operation resumes |
| Remote nuke | Admin via backend | Sends `nuke` command | Machine wipes NVS, reboots to setup screen |
| Factory reset | Owner at machine | Calls /v1/machine/factory-reset FIRST | Backend clears owner_id, generates new code, then NVS wiped |
| Factory reset offline | BLOCKED | Not allowed without network | Machine shows "Connect to internet to reset" |

**CRITICAL: Factory reset MUST call backend before wiping NVS.**
If NVS wiped without backend call → orphaned device in D1 → new owner cannot claim it.

---

## Nuke Command Security

The `nuke` command wipes the machine. It must be authenticated:
- Command delivered via /v1/machine/commands (GET, device_secret required)
- Machine checks X-Device-Secret on every command poll
- Backend only queues nuke for a specific device_id
- In PAYMENT_MODE=fake: device_secret check is relaxed for dev. In live: strict.

---

## Setup Code Generation

- Generated by backend (never by machine)
- Random 6 digits, stored in `setup_codes` table: `(code, assigned_mac, device_id, used, created_at)`
- `used` flag set on successful claim — code cannot be reused
- If machine is at factory before first /hello: no code exists yet. Code created on first /hello call.
- Lost sticker? Owner can retrieve code from dashboard (if already claimed: show device_id instead)

## Debug Mode (stolen machine recovery)

Gesture: hold bottom-left corner of screen for 5 seconds (no visual feedback during hold)
Shows: MAC address, device_id, firmware version, setup_code (if in NVS)
Purpose: law enforcement / support can identify stolen machine without sticker
No PIN required — this is recovery mode, deliberately accessible
Risk mitigation: 5s hold is hard to trigger accidentally during donation

---

## WiFi Credentials Security

- `config.h` WIFI_SSID and WIFI_PASS are COMPILE-TIME FALLBACK ONLY
- `config.h` MUST be in `.gitignore` — never commit real WiFi credentials to git
- Use `config.h.example` with placeholder values in repo
- At machine: owner can set WiFi via Settings tab → saved to NVS `nvs_ssid` / `nvs_pass`
- Boot priority: NVS WiFi first → config.h fallback if NVS empty

## Known Security Gaps (to fix before commercial launch)

| Severity | Gap | Fix |
|----------|-----|-----|
| HIGH | Factory reset orphans D1 | Add /v1/machine/factory-reset endpoint, require network for reset |
| HIGH | config.h WiFi in git | Add to .gitignore immediately |
| HIGH | No device_events audit table | Add D1 table: id, device_id, event_type, actor_id, timestamp, metadata |
| MED | Suspended device processes commands | Add suspended to auth rejection in authenticateDevice() |
| MED | Setup code duplicate MAC possible | Add UNIQUE constraint on setup_codes.assigned_mac |
| MED | Service PIN brute force | Implement 3-attempt lockout + 30s delay (planned, not built) |
| LOW | random() for lucky number | Replace with esp_random() (hardware RNG) |
| LOW | JTAG readable NVS in dev mode | Burn JTAG_DISABLE eFuse before production units ship |
| LOW | Debug mode gesture too visible | Consider 5s hold instead of tap pattern |

## Payment Gateway Modes

```
PAYMENT_MODE = fake         → fake QR, all 14 tests pass, no real money
PAYMENT_MODE = omise_test   → real scannable PromptPay QR, test keys, no real money
PAYMENT_MODE = omise_live   → real money — BLOCKED until KYC complete
```

Never set omise_live on a machine that is not physically present with hardware connected.
Switch via Cloudflare secret — never in code.

```sql
devices          — device_id, mac, owner_id, status, device_secret
setup_codes      — code, assigned_mac, device_id, used, created_at
users            — id, email, password_hash, role, created_at
device_events    — id, device_id, event_type, actor_id, timestamp, metadata  ← ADD THIS
```

`device_events` event_type values: `claimed`, `released`, `suspended`, `enabled`, `nuked`, `factory_reset`, `setup_code_generated`, `ownership_transferred`
